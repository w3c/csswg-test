CSS Font Style Matching Test

This test tests for steps 4. a-c of the font style matching algorithm. Stretch, style and weight information specified on the left-hand side should match the information on the right-hand side.

This test works by declaring @font-face rules for each combination of stretch, style and weight and refering to a different src url for each such rules. The font that the src url points to has a special property: it renders ligatures for the words "stretch", "style" and "weight" according to the values specified in its filename, e.g. CSSMatchingTest_condensed_normal_fourhundred.ttf renders the word/ligature "condensed" for the word stretch, "normal" instead of the word style, and "400" for the word weight. It also generates style declarations using IDs using the same naming scheme of stretch, style and weight.

The HTML then iterates over all the combinations and in each line refers to a new style. In the right column, the selected font can be verified by checking whether the correct ligatures were rendered. If they match, the user agent matched the right font-face rule and loaded the correct respective ttf file for rendering.
